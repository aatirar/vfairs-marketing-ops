# vFairs Mobile Event App — Capability Reference

**Type:** Platform Module (Attendee-facing + admin-configurable)
**Last updated:** 2026-05-22
**Source URLs:**
- Marketing page: https://www.vfairs.com/event-management-platform/mobile-event-app/

## 1. One-liner

A configurable mobile app (iOS + Android) that gives every attendee a personalized agenda, networking, push notifications, lead capture, and on-site check-in, available either inside the shared vFairs container app or as a fully white-labeled standalone app in the App Store and Google Play.

## 3. Capability Map

### 3.1 Attendee Experience
- **Personalized agenda** built from speakers, sessions, and tracks; supports saved seats and waitlists for capped sessions
- **Push notifications** for room changes, agenda updates, and emergencies (scheduled or sent in real time)
- **Interactive floor plans + seating charts** with one-tap directions to the next session
- **Browse, chat, and book meetings** with attendees, speakers, sponsors, and exhibitors
- **Digital badge** — the app itself acts as the badge for event and session check-in via QR codes
- **On-demand content + session replays** available 30 to 365 days post-event
- **Offline access** to agendas, maps, and saved session data

### 3.2 AI Capabilities
- **vFairs AI Event Assistant** (chatbot): answers attendee questions about sessions, speakers, parking, Wi-Fi, etc.
- **AI-generated personalized agendas** based on attendee role, interests, and registration data
- **AI-suggested networking matches** between attendees, speakers, sponsors, and exhibitors
- **AI session selection inside the mobile agenda** (admin can enable AI recommendations on the agenda view)

### 3.3 App Builder & Branding
- Drag-and-drop builder; no dev team required
- Toggle individual features on/off per event
- Full brand control: logo, colors, navigation layout
- **Dark mode** configurable per app
- **Force update** toggle so admins can require attendees onto the latest version

### 3.4 Navigation & Menu Management
- **My Event Menu** — event-specific menu navigation
- **Quick Links** (add/remove/manage) — v2 App Builder for shortcut tiles
- **QR scanner button** in bottom nav can be hidden if not needed
- **Splash advertisements** can be configured to fire when a menu item is tapped
- **Quick Links pop-ups + splash ads** can be tied to specific menu items

### 3.5 Event Information & Content
- **What's Happening section** — real-time event updates on the home screen
- **Image gallery** module with admin management
- **Auditorium image** is replaceable/removable from the mobile app
- **Privacy toggle** — admins can hide event location + analytics info from the attendee home page

### 3.6 Agenda & Sessions
- **Personalized schedule PDF export** — attendees can email themselves their saved schedule
- **AI-powered session selection** inside the mobile agenda
- **Custom filterable keyword tags** on sessions for attendee filtering
- **Speaker sorting** by last name option
- **Seat reservation status** can be surfaced in the attendee profile
- **Polls scoped to a specific session** can be created and surfaced in-session

### 3.7 Engagement & Gamification
- **Leaderboards + scavenger hunts**
- **Live polls, Q&A, and surveys**
- **Live event feed + social wall**
- **FAQs module** built into the app
- **Mobile app advertisements** — create promotional ads inside the app
- **Push notifications** — create, schedule, and target

### 3.8 Networking & Matchmaking
- **AI-suggested matches**
- **Chat + meeting booking** across attendees, speakers, sponsors, exhibitors
- **Video matchmaking** — one-on-one video meetings inside the app
- **Contact Notes pop-up** to capture notes when meeting another attendee
- **Live Event Feed permissions** — control which user types can post
- **Mobile-specific live feed permissions** layer

### 3.9 Sponsor & Exhibitor Value
- **Sponsors Hub** in mobile app for sponsor visibility
- **Sponsored video, splash banners, and push notifications** inside the app
- **QR-code lead capture** as a digital business-card replacement
- **Lead exports** to HubSpot, Salesforce, Marketo, or CSV
- **Meeting bookings** between sponsor reps and attendees on shared availability
- **Sponsor analytics** — session ratings, surveys, and booth visit data

### 3.10 Check-In & Registration
- **Admin check-in** — admins can check in users
- **Self check-in** — attendees can self-check
- **QR scan check-in** for admins
- **Add-on check-in** for supplementary items (meal tickets, workshops, swag)
- **Seat reservation status** display in profile
- Note: vFairs check-in supports QR + RFID smart badges + AI facial recognition (NFC is not supported)

### 3.11 User Management & Authentication
- **First-login password requirement** can be enforced
- **User role renaming** — customize role labels in the app
- **Profile Switcher** — one user can hold and switch between multiple profiles

### 3.12 Notes & Documentation
- **Product Notes** — attendees can add and manage product-specific notes
- **Note type visibility controls** — admins can hide note categories
- **Notes export/share** for attendees

### 3.13 Auction
- **Silent auction setup** inside the mobile app
- **Bid closure management** for ending auctions cleanly

### 3.14 Multilingual
- Menu items can be displayed in different languages (per attendee language preference)

### 3.15 Reporting & Analytics
- Real-time dashboards for session attendance, app usage, and engagement spikes
- Centralized tracking of attendee check-ins, session tap-ins, exhibitor badge scans
- Mobile-app-specific reports for engagement optimization

### 3.16 Security & Accessibility
- **SOC 2 Type II** certified
- **ISO 27001** certified
- **WCAG 2.1 AA** compliant
- **Current VPAT** available for accessibility reviews

## 4. Deployment Options

Two delivery modes — admins pick one per event:

| Mode | Time to ship | Where attendees download | Branding |
|---|---|---|---|
| **vFairs container app** | Days | Shared vFairs app on App Store + Google Play, entered via event code | Event branding inside a vFairs-branded shell |
| **White-labeled standalone app** | Longer (App Store / Play Store submission) | Custom listing on App Store + Google Play | Full white-label: event name, colors, logo, splash screen, branded notifications |

## 10. FAQ (verbatim from marketing page)

**Q: What is a vFairs Mobile Event App?**
All-in-one mobile solution designed to enhance the event experience for attendees, exhibitors, and organizers. Acts as a digital guide and a tool for event management, lead capture, networking, and attendee engagement.

**Q: Why would I need a mobile event app for my vFairs event?**
Creates a connected experience, reduces printing costs, streamlines logistics, boosts engagement for virtual and in-person attendees, bridges hybrid events.

**Q: How can a vFairs Mobile Event App save me money?**
Reduces or eliminates printing for programs, brochures, and signage. Provides sponsor ROI analytics that justify upsells.

**Q: Can the app be used for both virtual and in-person events?**
Yes. Offers on-site features (QR check-in, interactive maps) and virtual connectivity (chat, networking, live-streamed content).

**Q: How does lead capture work for exhibitors?**
Exhibitor scans an attendee's QR badge; lead info auto-populates and can be exported or synced to CRM.

**Q: What makes the vFairs Mobile Event App stand out from competitors?**
Integration with the broader vFairs platform, white-label option, full brand customization, QR check-in, 3D virtual environments, and one solution that covers hybrid events end-to-end.

**Q: Does your event app work offline?**
Yes. Offline access to agendas, maps, and saved session data.

**Q: What event app integrations do you offer?**
Salesforce, HubSpot, Marketo, Zoom, calendar integrations, lead retrieval systems, attendee data syncing.

**Q: What kind of support will I get for my event app?**
Dedicated project manager, 24/7 live chat during events, email support, Knowledge Base, Support Portal.

**Q: How do attendees download my mobile event app?**
Apple App Store or Google Play Store; container-app users enter an event code, white-label users open the branded app directly.

**Q: How do I build an event app?**
vFairs handles customization — branding, features, agenda, networking — through the app builder with project-manager support.

**Q: What is the process of getting an event app from you?**
Onboarding → content gathering → feature selection → build → test → App Store / Play Store submission (if custom-branded).

**Q: How much does an event app cost?**
Depends on event size, features, and customization. Quoted per engagement.

**Q: How long does it take to set up the app for my event?**
Varies by complexity. Container-app builds ship in days; white-label builds take longer due to store submission.
